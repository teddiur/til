Resumo

Tags semanticas HTML

HTML tem tags com significado, por isso semânticas. Elas servem como containers, assim como a tag genérica <div>, e portanto podem ser usadas para estruturar a página, mas também servem para tornar o código mais legível. Utiliza-las é uma boa prática para otimizar o site para o mecanismo de busca (Engine Optimization - SEO). A descrição de todas as tags semânticas pode ser consultada aqui. As principais tags semânticas estão explicadas a seguir:

<header>

É o cabeçalho da página, em geral contém o logo e um menu.

<main>

Contém o conteúdo principal da sua página e portanto não deve ter outras informações que são replicadas em outras páginas, como menu de navegação e logo.

<section>

Delimita uma seção do seu conteúdo. Pode ser utilizado para, por exemplo, separar introdução, produtos oferecidos e condições de compra em um site.

<article>

É um conteúdo específico, completo e independente. Geralmente tem texto e algum conteúdo multimidia

<nav>

Indica um menu de navegação, usualmente é um conjunto de tags <a>

<aside>

Utilizado quando se deseja destacar um trecho de um conteúdo ou artigo à parte, esse elemento também serve quando queremos chamar atenção para um dado adicional.

<footer>

Define o rodapé da página web. Coloca-se informações sobre o autor, direitos autorais, mídias sociais, política de privacidade, etc.

CSS

Também é uma linguagem de marcação, orientada a estilizar páginas HTML. CSS é abreviação de Cascading Style Sheets ou, em português, folhas de estilo em cascata. Há três formas de se utilizar o CSS: externamente, internamente ou inline.

Utiliza-se externamente quando cria-se um arquivo .css e no head do HTML há um link para esse arquivo. Internamente é quando as declarações de estilo estão dentro da tag <style> também dentro da head e inline é fazendo as declarações de estilo como o valor do atributo style. Isso pode ser feito em qualquer tag.
